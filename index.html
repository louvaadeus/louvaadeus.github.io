Como Baixar Vídeos do YouTube Usando o VLC Media Player

Em destaque

Posted on 14/09/2022 by louvaadeusinformatica
Responder

O VLC é uma multi-ferramentas de mídias digitais que pode ser executado em muitas plataformas (Linux, Windows, etc…), que pode reproduzir muitas mídias, como DVD e CD de vídeo e de áudio, como executor de streaming do Youtube entre outros. com diversas funcionalidades interessantes. Hoje veremos como baixar vídeos do Youtube usando esta ferramenta.

O VLC é um free software open source, multiplataforma muito popular como um players de mídia. Sua inteface simples pode fazer parecer que o software tem pouco a entregar, poucas pessoas sabem as vastas funcionalidaes que este software apresenta.

BAIXANDO VÍDEOS DO YOUTUBE

Para poder baixar vídeos do YouTube, temos que dar 5 passos:

1 – Copie a URL do vídeo escolhido no Youtube, marcando a URL e acionando o atalho de cópia CTRL + C.

Arrasta o cursor do mouse sobre a URL ela ficará como a imagem acima e após a URL marcara você aciona a tecla Ctrl fica segurando e a tecla C, copiando assim a URL co YouTube.

2 – Iniciar o VLC , acionar o menu Mídaicolar a URL no

Acionar o VLC através de seu ícone e uma vez aberto, escolher o menu Mídia e em seguida Abrir Transmissão de Rede.

Dentro da aba Rede, então voce cola a URL copiada dentro do quadro Por favor, preencha uma URL de rede:
E aciona para executar o vídeo dentro do VLC, quando o vídeo começar a tocar, dê pause na execução.

3 – Então na janela do VLC, clicar no menu Ferramentas e depois em Informações sobre o Codificador:

Aparecerá a janela Informações da Mídia Atual, no campo inferior da tela Local: dentro da caixa aparecerá a URL real do vídeo, marque-a arrastando mouse sobre ela e depois, acionando o atalho de cópia CTRL + C para copiar a URL do Vídeo Real.

4 – Abrir o navegador e colar a URL copiada para a URL do navegador, caso o navegador não comece a fazer o dowload em seguida, só clicar com a tecla direita do mouse sobre o vídeo no naveagador e aparecerá ,menu onde será escolhido Salvar vídeo como, escolhendo o diretório e o nome do vídeo que você vai gravar.

SWAP on-the-fly ou Memória SWAP no seu epaço de trabalho

Posted on 10/08/2023 by louvaadeusinformatica
Responder

No Linux é possível aumentar criar ou aumentar a sua Memória Swap, que é a memória auxiliar em disco para auxiliar a Memória RAM (memória principal do seu computador), quando esta fiscar com falta de espaço, com uma sequência de comandos bem simples, não precisa reparticionar seu disco físico, basta apenas criar um arquivo em sua unidade de disco (desde que haja espaço para isso).

Primeiro precisamos criar um arquivo que simulará a Memória RAM no disco chamada de SWAP on-the-fly, no meu caso vou criar um arquivo chamado SWAP-on-the-fly dentro do meu diretório home, com count blocos de tamanho bs (no caso, 8000 blocos de 1024k, ou seja, 8 gigabytes de memória swap:

dd if=/dev/zero of=SWAP-on-the-fly bs=1024k count=8000

Em seguida precisamos formatar este arquivo como swap (simula a memória):

mkswap SWAP-on-the-fly

Para concluir montamos o arquivo como memória swap:

swapon SWAP-on-the-fly

Observação:
Isso não torna essa memória swap permanente, após um novo boot da máquina, ela não estará mais ativa, para isso será preciso criar um script de configuração para ser rodado novamente o na inicialização do sistema.

Usado para dar apelido para um comando do Linuz

Suponha que eu queira uma lista de todos os tipos de arquivos que tenho em minha pasta atual (incluindo diretórios ,também chamados de pastas), vamos emitir o seguinte comando:

$ ls -lga

Para dar um apelido para esse comando, emito o comando ‘alias’, o nome do apelido ‘ea’ e logo depois do sinal de igual entre aspas coloco o comando ao qual esse apelido se refere ‘ls -lga’, dessa maneira:

$ alias ea=’ls -lga’

De agora em diante toda vez que eu comandar ‘ea’ o sistema operacional vai entender que quero emitir o comando ‘ls – lga’. Desta maneira:

$ ea

Exibirá como o resutado acima:

drwxr-xr-x 39 louvaadeus 4096 ago 1 13:54 .cache
drwxr-xr-x 2 louvaadeus 4096 fev 3 10:14 CastGroup
drwxr-xr-x 2 louvaadeus 4096 jul 18 18:26 Comandos
.
.

-rw——- 1 louvaadeus 53 ago 1 09:55 .Xauthority
-rw——- 1 louvaadeus 144749407 ago 1 14:58 .xsession-errors

No entanto, toda vez que eu deslogar do sistema, ele perde a lista de apelidos, para que este comando fique persistente no sistema, é necessário que você crie um aquivo de código que seja carregado no início do sistema como o bachrc.

Também é possível com o editor de texto criar um arquivo de nome ‘ea’ e inserir dentro deste arquivo o seguinte conteúdo:

#!/bin/bash
ls -lga

O arquivo começa com o shebang (#!/bin/bash) que executa o shell a partir do seu caminho original para que ele possa ser localizado não importa o caminho que o comando é executado, em seguida o comando que lista os arquivos (ls -lga).

Salve o arquivo e depois saia do editor. Ainda não está pronto, é necessário informar o Linux que este é um arquivo executável, com o seguinte comando:

$ chmod +x ea

Depois copiamos o arquivo ‘ea’ para o diretório ‘/usb/bin’, assim:

cp ea /usr/bin

Por fim, para executarmos o novo programa é só executar o comando ‘ea’ pois o mesmo já se encontra no path de execução numa instalação padrão do Linux.

$ ea

Louva a DEUS Linux (Debian 11) + Docker + Compose + Ruby on Rails + PostgreSQL

Posted on 13/04/2023 by louvaadeusinformatica
Responder

1 – Instalar ambiente Docker no Louva a DEUS Linux

1.1 – Atualizar o Louva a DEUS Linux

Abrir o terminal pelo ícone adequado

ou então através do atalho de teclado Ctrl + Alt + T, logo em seguida atualizar a lista de pacotes do repositório do Linux com o comando sudo apt update (informe a senha se necessário), depois atualizar todos os pacotes com sudo apt upgrade, informe S ou Y quando necessário:

$ sudo apt update

$ sudo apt upgrade

Desta forma temos nosso ambiente de sistema bem atualizado.

1.2 – Testar se o “Docker” e o “Docker Compose” estão instalados

Comandos para ver as versões do Docker e do Docker Compose:

$ docker –version

$ docker-compose –version

O comando que te mostrar a versão é porque está instalado, aquele que não apresentar a versão, tem que instalar com os comandos do ítem 1.3.

1.3 – Instalar o “Docker” e o “Docker Compose“

Agora vamos instalar o ambiente do Docker, temos dois tipos de pacotes para instalar, o Docker EE – Docker Enterprise Edition e o Docker CE – Docker Community Edition, um é pago e o outro é gratuíto, utilizaremos o Docker CE que é gratuíto, com os seguintes comandos:

$ sudo apt install docker-ce -y

$ sudo apt install docker-compose -y

Então emitimos novamente os comandos para mostrar as versões do Docker e Docker Compose:

$ docker –version

$ docker-compose –version

Se os dois comandos mostrarem as versões, estamos preparados para usar Docker com Docker Compose.

1.4 – Permitir que seu “Usuário Comum” possa executar o “Docker” (sem ser o usuário Root)

Para permitir que nosso Usuário Comum possa executar o Docker, sem que seja necessário acionar o usuário Root ou usar o comando sudo:

1.4.1 – Em primeiro lugar criamos o “Grupo de Usuários” de nome “docker” com o comando:

$ sudo groupadd docker

1.4.2 – Em seguida adicionamos nosso “Usuário Comum” (que estamos usando no terminal de linha de comando) ao Grupo de Usuários “docker” com o comando:

$ sudo usermod -aG docker $USER

Obs importante: para que o Usuário Comum passe a ser considerado como participante do grupo docker é necessário logar como Usuário Comum novamente (re-logar) ou então reiniciar o Linux (solução menos Hacker … hehehe)

1.4.3 – Para verificarmos o “Usuário Comum” adicionado ao grupo “docker” usamos o comando:

$ cat /etc/group

No caso, meu Usuário Comum é o usuário louvaadeus.

Com todos esses comandos executados, estmos prontos para orquestrar nosso Docker com o Usuário Comum.

Dicas: Lugar legal para achar contêiners (Docker Hub)
“https://hub.docker.com/search?image_filter=official&type=image&q=&page=1“

1.4.4 – Help, sempre é muito interessante saber obter o manual da ferramenta

$ docker help

2 – Orquestrar o contêiner com Linux, Ruby on Rails, PostgreSQL no Docker com Docker Compose

2.1 – Carregar o diretório do seu Usuário Comum (não Root)

$ cd ~

2.2 – Criar o diretório “/rails” para as aplicações em Ruby on Rails

$ mkdir rails

2.3 – Usar o diretório “/rails” como padrão para suas aplicações em Rails

Neste caso é só uma questão de organização, para que suas aplicações Rails estejam em um mesmo grupo de diretórios.

$ cd rails

2.4 – Listar imagens do Docker

Para listar as imagens que tem no seu servidor Docker, execute o seguinte comando:

$ docker image ls

Se em sua máquina ainda não existe contêiners, aparecerá para você uma lista como essa de minha máquina, uma lista vazia, porque ainda não foram criados contêiners, mais adiante, depois da criação você verá uma lista com máquinas exibidas na lista.

3 – Como criar uma aplicação Ruby on Rails com PostgreSQL, chamada jmrpsge no Compose

Referência: “https://github.com/docker/awesome-compose/tree/master/official-documentation-samples/rails/“

Este guia de início rápido mostra como usar o Docker Compose para configurar e executar um aplicativo Rails / PostgreSQL

3.1 – Crie um diretório específico para sua aplicação jmrpsge dentro do grupo de diretórios rails

3.1.1 – Torne o diretório anteriormente criado rails seu diretório de trabalho

$ cd ~/rails

3.1.2 – Crie o diretório de sua aplicação, no meu caso jmrpsge

$ mkdir /jmrpsge

Obs: jmrp vem do nome de minha empresa, cujo nome fantasia é Louva a DEUS Informática e sge significa Sistema Gerenciador de Empresas, um ERP (Enterprise Resource Planning) que contém os sistemas integrados que uma empresa necessita.

3.2 – Torne jmrpsge seu diretório de trabalho

$ cd ~/rails/jmrpsge

3.2 – Defina o projeto

Comece configurando os arquivos necessários para criar o aplicativo. O aplicativo será executado dentro de um contêiner Docker contendo suas dependências. A definição de dependências é feita usando um arquivo chamado Dockerfile.

3.2.1 – Para criar, o “Dockerfile” emita o seguinte comando:

$ touch Dockerfile

Insira o seguinte texto dentro do arquvio Dockerfile com o editor de textos de sua preferência.

syntax=docker/dockerfile:1
FROM ruby:2.5
RUN apt-get update -qq && apt-get install -y nodejs postgresql-client
WORKDIR /jmrpsge
COPY Gemfile /jmrpsge/Gemfile
COPY Gemfile.lock /jmrpsge/Gemfile.lock
RUN bundle install
# Add a script to be executed every time the container starts.
COPY entrypoint.sh /usr/bin/
RUN chmod +x /usr/bin/entrypoint.sh
ENTRYPOINT [“entrypoint.sh”]
EXPOSE 3000
#Configure the main process to run when running the image
CMD [“rails”, “server”, “-b”, “0.0.0.0”]

Isso colocará o código do aplicativo dentro de uma imagem que cria um contêiner com Ruby, Bundler e todas as suas dependêcias dentro dele. Para obter informações sobre escrever Dockerfiles, consulte o guia do usuário do Docker (https://docs.docker.com/get-started/https://docs.docker.com/get-started/) e a referência do Dockerfile (https://docs.docker.com/engine/reference/builder/).

3.2.2 – Em seguida, abra um editor e crie um bootstrap “Gemfile” que apenas carrega o Rails. Isso será substituído em um momento por “rails new”.

$ touch Gemfile

E colar dentro dele o texto abaixo:

source ‘https://rubygems.org&#8217;
gem ‘rails’, ‘~>5’

3.2.3 – Crie um arquivo vazio “Gemfile.lock” para construir nosso arquivo “Dockerfile”

$ touch Gemfile.lock

3.2.4 – Em seguida, forneça um script de ponto de entrada para corrigir um problema específico do Rails que impede o servidor de reiniciar quando um determinado arquivo “server.pid” já existe. Este script será executado sempre que o contêiner for iniciado.

Criar o arquivo entrypoint.sh:

$ touch entrypoint.sh

#!/bin/bash
set -e
# Remove a potentially pre-existing server.pid for Rails.
rm -f /jmrpsge/tmp/pids/server.pid
# Then exec the container’s main process (what’s set as CMD in the Dockerfile).
exec “$@”

3.2.5 – Finalmente, “docker-compose.yml” é onde a mágica acontece.

Este arquivo descreve os serviços que compõe seu aplicativo (um banco de dados e um aplicativo Web), como obter a imagem do Docker de cada um (o banco de dados é executado apenas em uma imagem PostgreSQL pré-fabricada e o aplicativo da Web é criado a partir do diretório atual), e a configuração necesária para vinculá-los e expor a porta do aplicativo da web.

services:
  db:
    image: postgres
    volumes:
      - ./tmp/db:/var/lib/postgresql/data
    environment:
      POSTGRES_PASSWORD: password
  web:
    build: .
    command: bash -c "rm -f tmp/pids/server.pid && bundle exec rails s -p 3000 -b '0.0.0.0'"
    volumes:
      - .:/jmrpsge
    ports:
      - "3000:3000"
    depends_on:
      - db

Dica: Você pode usar uma extensão .yml ou .yaml para este arquivo.

4 – Construa o projeto

4.1 – Com esses arquivos no lugar, agora você pode gerar o aplicativo de esqueleto do Rails usando o “docker compose run“:

$ docker compose run –no-deps web rails new . –force –database=postgresql

Primeiro o Compose cria a imagem para o serviço web usando o arquivo Dockerfile.

O -no-deps diz ao Compose para não iniciar serviços vinculados. Em seguida, ele é executado rails new dentro de um novo contêiner, usando essa imagem. Uma vez feito isso, você deve ter gerado um novo aplicativo.

Liste os asrquivos:

4.2 – Se você estiver executando o Docker no Linux (meu caso), os arquivos rails new criados serão de propriedade do “root”.

Isso acontece porque o contêiner é executado como usuário raiz. Se for esse o caso, altere a propriedade de novos arquivos (pode solicitar a senha para o sudo):

$ sudo chown -R $USER:$USER .

Assim você pode trabalhar a aplicação como Usuário Comum.

Se você estiver executando o Docker no Mac ou no Windows, você já deve ter a propriedade de todos os arquivos, incluindo aqueles gerados por rails new.

4.4 – Conecte o banco de dados

O aplicativo agora é inicializável, mas você ainda não chegou lá. Por padrão, o Rails espera que um banco de dados esteja rodando localhost então, você precisa apontá-lo para o conteiner db.Você também presica alterar o banco de dados e o nome do usuário para alinhar com os padrões definidos pela imagem postgres.

Substitua o conteúdo do config/database.yml pelo seguinte:

default: &default
adapter: postgresql
encoding: unicode
host: db
username: postgres
password: password
pool: 5
development:
<<: *default
database: myapp_development
test:
<<: *default
database: myapp_test

4.5 – Agora você pode inicializar o aplicativo com o “docker compose up“.

Se tudo estiver bem, você deverá ver alguma saída do PostgreSQL:

$ docker compose up

4.6 – Finalmente, você precisa criar o banco de dados. Em outro terminal, execute:

$ docker compose run web rake db:create

5 – Veja a página de boas-vindas do Rails

É isso. Seu aplica agora deve estar em execução na porta 3000 em deu daemon Docker.

No Docker Desktop para Mac e no Docker Desktop para Windows, acesse http://localhost:3000 em um navegador da Web para ver o Rails Welcome.

6 – Pare o aplicativo

Para interromper o aplicativo, execute docker compose down no diretório do projeto. Você pode usar a mesma janela de terminal na qual iniciou o banco de dados ou outra em que tenha acesso a um prompt de comando. Esta é uma maneira limpa de parar o aplicativo.

$ docker compose down

7 – Reinicie o aplicativo

Para reiniciar o aplicativo, execute o docker compose up no diretório do projeto.

$ docker compose up

8 – Recrie o aplicativo

Se você fizer alterações no Gemfile ou no arquivo Compose para experimentar algumas configurações diferentes, será necessário reconstruí-lo. Algumas alterações requerem apenas docker copose up –build, mas uma reconstrução completa requer uma nova execução do docker compose run web bundle install para sincronizar as alterações host Gemfile.lock, sequida por docker compose up –build.

Aqui está um exemplo do primeiro caso, onde uma reconstrução completa não é necessária. Suponha que você simplemente queira alterar a porta exposta no host local de 3000 nosso primeiro exemplo para 3001. Faça alteração no arquivo Compose para expor a porta 3000 no contêiner por meio de uma nova porta, 3001, no host e salve as alterações:

ports:
-“3001:3000”

Agora, reconstrua e reinicie o aplicativo com docker compose up –build.

Dentro do contêiner, seu aplicativo está sendo executado na mesma porta de antes 3000, mas o Rails Welcome agora está disponível http://localhost:3001 em seu host local.

Instalar o PostgreSQL no Louva a DEUS Linux (Debian 11)

Posted on 05/04/2023 by louvaadeusinformatica
Responder

1 – Recomenda-se atualizar a lista de pacotes e instalar os pacodes de upgrade sugeridos anteriormente

$ sudo apt update && sudo apt upgrade

2 – Instalar o PostgreSQL

$ sudo apt install postgresql postgresql-contrib

3 – Verificar como está o serviço do PostgreSQL

$ sudo systemctl status postgresql

4 – Você também pode para e iniciar o serviço do PostgreSQL, com os seguintes comandos:

4.1 – Parar o serviço do PostgreSQL em execução

$ sudo systemctl stop postgresql

4.2 – Iniciar o serviço do PostgreSQL em parado

$ sudo systemctl start postgresql

5 – Configurara a senha no PostgreSQL

$ sudo passwd postgres
